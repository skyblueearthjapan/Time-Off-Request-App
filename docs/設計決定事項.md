# 設計決定事項（2026-03-04 確定）

## 1. 権限KEY名の統一

| KEY | 用途 | 備考 |
|-----|------|------|
| `ALLOW_ADMIN_PAGE_EMAILS` | 管理ページ認可リスト（**真実**） | 管理者＋総務を列挙 |
| `MAIL_TO` | 通知先 | 管理者＋総務 |
| `SOMU_EMAILS` | 総務メンバー管理（参考情報） | 認可判定には使わない |
| `GENERAL_EMAILS` | 申請ページ利用者（必要時追加） | 未指定＝ドメイン内全員 |

**原則**: 認可の真実は `ALLOW_ADMIN_PAGE_EMAILS` に集約。

## 2. STATUS値

### 初期運用（画面に出す）
- `申請中`
- `承認済`

### DB定義のみ（将来拡張用、画面には出さない）
- `取消` — 申請後の取り下げ
- `却下` — 記載不備で出し直し

**方針**: "今は運用しないが、DBでは定義だけ残す"

## 3. 管理ページ構成

- **1画面構成**: `admin.html` 内でタブ/ボタン切替
  - 「上長ビュー」— 管轄部署の作業員別取得状況
  - 「総務ビュー」— 全部署・全作業員・フィルタ・アラーム
- 権限による表示制御:
  - 総務/管理者 → 両方見える
  - 上長 → 上長ビューのみ

## 4. マスタ同期

- **同期元**: 外部スプレッドシート（会社の正マスタ）
- **同期先**: 休暇届アプリDB の `M_DEPT` / `M_WORKER`
- CONFIGに追加するKEY:
  - `MASTER_SOURCE_SSID` — 同期元スプレッドシートID
  - `MASTER_SOURCE_DEPT_SHEET` — 部署シート名
  - `MASTER_SOURCE_WORKER_SHEET` — 作業員シート名

## 5. プレビュー・承認の画面遷移

**別ページ遷移方式を採用**

```
TOP（申請モーダル）
  ↓ 入力確定
preview.html（紙様式を全面表示）
  ↓ 承認ボタン押下
sign.html（サイン欄を大きく表示、canvas）
  ↓ 完了
TOPへ戻る
```

**理由**:
- 紙様式は"画面いっぱい"が要件 → モーダルでは狭い
- スマホ/タブレット対応が楽
- canvas のサイズ制御が安定

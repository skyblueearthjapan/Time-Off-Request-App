# ✅ 休暇届申請システム 要件定義（確定版）

## 1. システム全体構成

### ■ 技術構成
- フロント：GAS Webアプリ（HTMLService）
- バックエンド：Apps Script
- DB：Googleスプレッドシート
- PDF：スプレッドシートテンプレート差し込み → PDF化
- 保管：指定Google Driveフォルダ
- 通知：GmailApp

## 2. 承認方式（重要確定事項）

### ■ 承認者の特定は不要
- 作業員が上長の元へ行く
- その場で上長が画面を確認
- サインを書く
- サイン＝承認

### ■ 保存する情報
- 承認日時
- サイン画像
- 承認状態（承認済）
- **承認者アカウント管理は不要（シンプル設計）**

## 3. ログイン・権限設計

### ■ Googleアカウントで制御

| 区分 | アクセス可能ページ |
|------|-------------------|
| 一般利用 | 申請ページのみ |
| 総務 | 管理ページ＋全作業員閲覧 |
| 管理者 | 全ページ |

### ■ 権限管理方法
スプレッドシートに設定シートを作成：

**M_SYSTEM_SETTING**

| KEY | VALUE |
|-----|-------|
| ADMIN_EMAILS | aaa@gmail.com,bbb@gmail.com |
| GENERAL_EMAILS | xxxx@gmail.com |
| MAIL_TO | 管理者メール,総務メール |
| PDF_FOLDER_ID | DriveフォルダID |

→ ここを書き換えれば運用変更可能

## 4. 申請画面仕様

### 4-1 TOP画面
- **上部：**
  - 部署選択
  - 作業員選択（部署連動）
- **中央：**
  - 「休暇届申請」横長ボタン
- **下部：**
  - これから休暇予定の一覧表示
  - 本日以降
  - 当日分は0:00で非表示

## 5. 申請モーダル仕様（入力設計）

### ① 休暇区分（必須）
- 全日休
- 半日休

### ② 休暇日（必須）

### ③ 半日の場合のみ表示
- 午前
- 午後

### ④ 休暇種類（条件分岐）

**● 全日休**
- 有給消化（私用）
- 振替休暇
- 特別休暇

**● 半日休**
- 有給消化（私用）
- 特別休暇

### ⑤ 振替休暇の場合
- 振替元出勤日（必須）

### ⑥ 特別休暇の場合
- 参考例表示（結婚・忌引等）
- 任意テキスト入力

### ⑦ 有給の場合
- 任意詳細入力
- 病欠時は詳しく記載

### ⑧ 詳細追加入力
- 「詳細を追加しますか？」
- Yes → 自由記述欄表示
- Skip可能

### ⑨ 申請確定
→ プレビュー画面へ

## 6. プレビュー・承認画面（肝部分）

### ■ 紙様式をWeb全面表示
- スプレッドシートテンプレをHTML再現
- 入力済内容を差し込み表示

### ■ 承認ボタン押下
→ サイン画面に遷移

### ■ サイン仕様
- canvas使用
- タッチ/ペン対応
- 画像保存（Base64→Drive保存）

### ■ 承認完了後
- ステータス更新
- 承認日時保存
- PDF生成
- Drive保存
- メール通知

## 7. PDF生成仕様

### ■ 方法
1. PDFテンプレ用スプレッドシートをコピー
2. セルに値を書き込み
3. PDFエクスポート
4. Drive指定フォルダへ保存
5. URLを申請データに保存

## 8. 有給取得義務チェック設計（超重要）

### ■ 年度定義
- 4/1〜3/31

### ■ カウント対象
- 「有給消化（私用）」のみ

### ■ 警告基準

| 期限 | 必要回数 |
|------|---------|
| 6月末 | 1回 |
| 8月末 | 2回 |
| 12月末 | 3回 |
| 2/15 | 4回 |
| 年度末 | 5回 |

### ■ 警告動作

**① 画面表示**
- 未達者を赤表示
- アラートマーク表示

**② メール通知**
- 管理者
- 総務

### ■ 送信タイミング（確定）
- 各期限の月末または月初
- → 月初送信が安全（例：7/1, 9/1, 1/1, 2/16）
- 時間トリガーで実装

## 9. 管理ページ仕様

### 9-1 上長用
- 部署選択モーダル
- 管轄部署のみ表示
- 作業員別年度取得状況表示
- 有給取得回数表示
- 未達警告表示

### 9-2 総務ページ
- 全部署表示
- 全作業員表示
- フィルタ機能
- 有給回数列あり
- 未達者アラームあり

## 11. 次のステップ

- **A. 画面遷移図を作る** — TOP → モーダル → プレビュー → サイン
- **B. シート構造を具体的に設計する** — 列定義を完全確定させる
- **C. トリガー設計** — マスタ同期・警告判定・メール送信

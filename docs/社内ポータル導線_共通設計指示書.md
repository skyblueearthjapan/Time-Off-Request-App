# 社内ポータルサイト導線ボタン — 共通設計指示書

> **対象**: LINEWORKS 社内ポータルサイトから起動される全GASアプリケーション
> **目的**: 各アプリケーションからポータルサイトへ戻るナビゲーションを統一仕様で実装する
> **作成日**: 2026-03-03
> **先行実装**: 残業・休日出勤申請アプリ（実装済み・本番稼働中）

---

## 1. 概要

社内ポータルサイト（下記URL）をハブとして、各アプリケーションを別タブで開く構成になっている。
ユーザーが各アプリケーションからポータルに戻れるよう、**全画面のヘッダーナビゲーション**に
「社内ポータルへ移動」ボタンを設置する。

## 4. 実装仕様

### 4-1. バックエンド（GAS側）

`doGet(e)` 関数内で、HTMLテンプレートに `PORTAL_URL` 変数を渡す。

```javascript
const t = HtmlService.createTemplateFromFile('画面名');
t.APP_URL = ScriptApp.getService().getUrl();
t.PORTAL_URL = 'https://script.google.com/a/macros/lineworks-local.info/s/AKfycbx2eyJMOYP9o--GPBuhY-pj071IIR6Kqb_0xALwwNzdLQZux0dIAlL3P9EoCucnzXA/exec';
```

**注意点:**
- テンプレートを生成する**全箇所**で `t.PORTAL_URL` を設定すること
- エラー画面（権限なし等）のテンプレートにも忘れず設定すること
- URLはハードコードで問題ない（ポータルのデプロイURLは固定）

### 4-2. フロントエンド（HTML側）

#### A. 通常画面 — ヘッダーナビゲーション内に配置

既存のナビリンク（「トップへ」「承認者画面」等）の**末尾（右端）**に追加する。

```html
<header>
  <div>
    <div class="title">アプリ名</div>
    <div class="sub">サブタイトル</div>
  </div>
  <div class="row">
    <!-- 既存のナビリンク群 -->
    <a href="<?= APP_URL ?>?page=xxx" target="_top" class="nav-link">○○画面</a>
    <!-- ▼ これを末尾に追加 ▼ -->
    <a href="<?= PORTAL_URL ?>" target="_top" class="nav-link portal-link">社内ポータルへ移動</a>
  </div>
</header>
```

#### B. エラー画面（権限なし等）— ボタンとして配置

```html
<a href="<?= APP_URL ?>?page=top" target="_top" class="btn">トップ画面へ戻る</a>
<div style="margin-top:12px;">
  <a href="<?= PORTAL_URL ?>" target="_top" class="btn"
     style="background:linear-gradient(135deg, #1f9d55, #4ade80);">社内ポータルへ移動</a>
</div>
```

### 4-3. CSS — ポータルリンク専用スタイル

各HTMLの `<style>` セクション内、`.nav-link:hover` の直後に追加する。

```css
/* --- 既存 --- */
.nav-link{
  font-size:13px; padding:6px 12px; border-radius:8px;
  border:1px solid var(--line); background:#fff; color:var(--text);
}
.nav-link:hover{ background:var(--bg1); text-decoration:none; }

/* --- ▼ 追加 ▼ --- */
.portal-link{ background:#1f9d55; color:#fff; border-color:#1f9d55; }
.portal-link:hover{ background:#177a43; color:#fff; }
```

**デザイン仕様:**

| 項目 | 値 | 備考 |
|------|----|------|
| 背景色（通常） | `#1f9d55` | ポータルサイトのブランドグリーン |
| 背景色（hover） | `#177a43` | やや暗いグリーン |
| 文字色 | `#ffffff` | 白 |
| ボーダー色 | 背景色と同色 | |
| フォントサイズ | `13px` | 既存 nav-link と統一 |
| パディング | `6px 12px` | 既存 nav-link と統一 |
| 角丸 | `8px` | 既存 nav-link と統一 |
| ラベル文言 | `社内ポータルへ移動` | 全アプリ統一（変更不可） |

> **CSS変数について**: アプリによって `--accent` の値が異なる可能性があるため、
> ポータルリンクは CSS変数ではなく **直値 `#1f9d55`** を使用すること。

## 5. 配置ルール

```
┌─────────────────────────────────────────────────────┐
│ header                                               │
│  [タイトル]        [画面A] [画面B] [社内ポータルへ移動] │
│  [サブタイトル]     ← 白背景  白背景   緑背景(末尾固定) │
└─────────────────────────────────────────────────────┘
```

- ポータルリンクは常に**ナビリンク群の最右端**に配置
- 他のナビリンク（白背景）と差別化するため**緑背景・白文字**
- `target="_top"` 必須（GAS iframe から脱出するため）
- レスポンシブ時はナビリンクが折り返して縦並びになる想定で問題なし

## 6. チェックリスト

実装時に以下を全て確認すること:

- [ ] `doGet` 内の**全テンプレート生成箇所**で `t.PORTAL_URL` を設定している
- [ ] エラー画面のテンプレートにも `t.PORTAL_URL` を設定している
- [ ] 全画面のヘッダーにポータルリンクを追加している
- [ ] ポータルリンクはナビリンク群の**末尾**に配置している
- [ ] CSSクラス名は `.portal-link`（統一）
- [ ] ボタンラベルは「社内ポータルへ移動」（統一）
- [ ] `target="_top"` が設定されている
- [ ] 背景色は直値 `#1f9d55` を使用（CSS変数ではない）
- [ ] デプロイ後、全画面でボタンが表示され遷移できることを確認
